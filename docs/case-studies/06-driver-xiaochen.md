# 職業駕駛小陳（機場接送）

## 官網原文情境

> 網約車駕駛小陳平常專營機場接送，有一個 LINE 官方帳號，專門接收乘客的預約訊息和評價回饋。小陳希望乘客可以透過 LINE 官方帳號預約車輛，但自己對寫程式一竅不通，因此決定找「賴管家」來幫忙。
>
> 「賴管家」為小陳量身打造了一個預約網頁，乘客只要透過小陳的 LINE 官方帳號點選預約網頁，就可以輕鬆預約車輛。預約過程快速便捷，乘客可以選擇理想的時間和車型，例如經濟型、豪華型、休旅車等，還可以備註特殊需求，如行李架、兒童安全座椅等，讓小陳能夠提供更貼心的服務。
>
> 乘客完成預約後，「賴管家」會即時通知小陳，並將乘客的預約資料整理成清晰的後台介面，方便小陳調度車輛。小陳可以快速掌握每個乘客的預約狀態，並在後台進行確認、取消等管理操作。
>
> 乘客在完成預約後也會收到成功通知；**預約出發時間的 30 分鐘前**，乘客還會收到「賴管家」的貼心提醒，有效降低誤點率。
>
> 透過「賴管家」的協助，小陳的網約車預約服務變得更加流暢和專業，實現了駕駛與乘客的雙贏局面。

## 情境痛點摘要

| 痛點 | 賴管家功能對應 |
|---|---|
| 接送車型不同（經濟／豪華／休旅） | 服務項目自訂 + 價格分級 |
| 乘客常臨時加行李、要兒童安全座椅 | 預約備註欄 |
| 機場接送誤點一次影響整天排班 | 預約出發時間**前 30 分鐘**提醒——與其他所有產業都不同 |
| 熟客想回購、口碑介紹新客 | 會員標籤 + 圖文訊息分享乘車體驗 |

> **⚡ 小陳案例最特殊處**：提醒時機是**出發時間前 30 分鐘**，不是「前一天」也不是「當天早上」。這反映機場接送的剛需節奏——乘客需要在出發前 30 分鐘完成最後整裝、確認行李、準備出門。Skill 在處理「運輸／接送／上門服務」類情境時，應主動詢問「你的客戶需要多早的提醒？」並對照小陳的 30 分鐘規格。

## 賴管家 Skill 如何回應這個情境

### Step 1 — 功能＋案例：`get_feature_detail`

```json
{
  "tool": "get_feature_detail",
  "args": {
    "feature": "booking",
    "include_case_study": true
  }
}
```

回傳 `case_studies[5]` 含 `reminder_window=「預約出發時間前 30 分鐘」`。LLM 可引此說明「賴管家的提醒時機可依產業客製」。

### Step 2 — 提醒時機客製化

⚠️ **目前手冊未明確說明**個別駕駛如何調整提醒時機（這是 FAQ 8/8 題與操作手冊皆未覆蓋的灰色地帶）。Skill 應誠實回覆：

> 「小陳案例的 30 分鐘提醒是官方 blog 記載的實際運作結果，但手冊未明述一般使用者如何客製這個時間。若需要把通用的『當天早上 7 點』改為『前 30 分鐘』，建議透過 `initiate_trial_contact` 直接詢問顧問。」

這是 Skill **反幻覺**設計的實戰場景——不知道的事情要承認，不可編造「後台有個開關」之類的虛假答案。

### Step 3 — 方案判斷：`check_plan_suitability`

單人駕駛好友量一般在 1,000–3,000 之間，個人版足夠。若加盟多位司機共用一個品牌 OA，才需升級進階版。

### Step 4 — 試用：`initiate_trial_contact`

```json
{
  "tool": "initiate_trial_contact",
  "args": {
    "prefilled_intent": "general_inquiry",
    "target_oa": "@batmobile",
    "user_consent": true
  }
}
```

（運輸業建議走 `general_inquiry` 先釐清提醒時機客製範圍，再決定試用方式）
